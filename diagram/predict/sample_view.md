# 価格予測表示サンプル

## 概要
このサンプルでは、非同期予測 API を使用して価格予測を行い、結果をフロントエンドに表示する流れを実装します。

## 基本的な流れ
1. 2日分の価格情報を API から取得する
2. 前半1日分の情報を `predict_with_polling` に与えて次の24時間を予測させる（非同期処理）
3. 実際の後半1日分の価格と予測結果を比較表示する

## 実装手順

### 1. データ取得インターフェース
- 期間を指定して価格情報を取得するためのUIを実装
  - 開始日時と終了日時の選択
  - トークンペアの選択

### 2. 予測処理
- 取得したデータの前半を使って `predict_with_polling` を呼び出す
  - 非同期予測処理でポーリングによりプログレス表示
  - 予測期間は取得データの後半と一致させる
  - オプションでモデル名やパラメータを指定可能にする

### 3. 結果表示
- 実際の価格データと予測結果を同一グラフに表示
  - 実線：実際の価格
  - 点線：予測価格
- 予測精度の評価指標を表示
  - RMSE（二乗平均平方根誤差）
  - MAE（平均絶対誤差）
  - MAPE（平均絶対パーセント誤差）

## 利用する主要コンポーネント
- `chronos_api::predict::ChronosApiClient` - 予測APIクライアント
- `stats.rs` の既存コンポーネント - データ取得と表示の基盤として活用
- `chart::plots` モジュール - 予測結果と実際のデータの可視化

## 注意点
- 予測処理は非同期で行い、ポーリングによってプログレス表示する
- 長時間の予測処理でもユーザーインターフェースをブロックしない
- データ量が多い場合の処理速度とメモリ使用量の最適化
- エラーハンドリングを適切に実装して、ユーザーにわかりやすく表示
